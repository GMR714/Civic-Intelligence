**Gathered:** 2026-04-13
**Status:** Ready for planning

<domain>
## Phase Boundary

Build the high-fidelity bento-box UI and dashboard layouts using static mock data to validate the platform's visual presentation and concept flow, without implementing backend or interactive validation logic yet.

</domain>

<decisions>
## Implementation Decisions

### Layout Scope
- **D-01:** Implement a multi-page navigable structure rather than a single continuous scroll ("Pitch Deck") page. Must include at least the Main Dashboard and a Proposal Detail view.

### Visual Complexity 
- **D-02:** Prioritize high-end aesthetics (glassmorphism bento boxes, deep dark mode, bioluminescent accents) using static CSS and Tailwind rather than live interactive charting libraries like Recharts for now to speed up prototyping.

### Mock Data Content
- **D-03:** Content should be context-specific and localized to "Ipe City" / "Florianópolis" (e.g., real-sounding urban proposals, recognizable local issues) rather than generic "Lorem Ipsum".

### the agent's Discretion
- **Design of the Impact Matrix UI:** (e.g., whether to use side-by-side lists, radar charts, or simple colored tags).
- **Navigation Menu Design:** Sidebar, top navigation, or floating dock.
- **Specific Micro-animations:** (using framer-motion) to make the static UI feel premium.

</decisions>
